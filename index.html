SISTEMA DE GESTIÓN ISHAJON

Automatización de Flujos de Trabajo de RRHH

1. DESCRIPCIÓN DEL SISTEMA

Aplicación web integral diseñada para digitalizar el proceso de solicitud, aprobación y seguimiento de permisos laborales. Reemplaza procesos manuales con un flujo electrónico estructurado.

2. ARQUITECTURA TÉCNICA (BACKEND & FRONTEND)

  • Backend: PHP 8.1 con Framework Laravel 11.x. Uso de Eloquent ORM para manejo de datos y Policies para seguridad.
  • Base de Datos: PostgreSQL 17.4. Estructura relacional de 8 tablas principales (Empleados, Solicitudes, Aprobaciones, etc.).
  • Frontend: Blade Templates con Laravel Livewire para reactividad. Bootstrap 5 para UI responsiva.
  • Seguridad: Autenticación robusta, Firma Digital para aprobaciones y protección CSRF/XSS.

3. FLUJO DE NEGOCIO IMPLEMENTADO

  • Etapa 1 (Solicitud): Empleado genera solicitud con tipo, fecha y justificación.
  • Etapa 2 (Aprobación Nivel 1): Director revisa, define compensación/descuento y firma digitalmente.
  • Etapa 3 (Aprobación Nivel 2): Gestión Humana valida normatividad y da el cierre final.
  • Notificaciones: Sistema automático de alertas por correo en cada cambio de estado.

4. IMPACTO

Reducción de tiempos de respuesta administrativos, trazabilidad 100% de las solicitudes para auditoría y generación de reportes estadísticos de ausentismo.

ANÁLISIS DE REQUERIMIENTOS (SRS)

Muestra de Estructuración Técnica y Funcional

Este documento demuestra la capacidad para traducir necesidades de negocio en especificaciones técnicas claras para el equipo de desarrollo.

REQ 1: FUNCIONALIDAD CORE

  • Funcional: El sistema debe permitir al usuario [acción] de manera segura y controlada.
  • Técnica: Implementación de módulo que procesa datos, valida reglas de negocio y almacena en base de datos retornando respuesta en tiempo óptimo.

REQ 2: GESTIÓN DE USUARIOS (RBAC)

  • Objetivo: Controlar acceso a funcionalidades según perfil.
  • Solución Técnica: Sistema de autenticación con encriptación de contraseñas, control de sesiones y middleware de autorización para validar permisos por rol.

REQ 3: REPORTES Y DATOS

  • Funcional: Generación de reportes exportables (PDF/Excel) para toma de decisiones.
  • Técnica: Consultas optimizadas a fuente de datos, procesamiento con filtros dinámicos y renderizado gráfico en frontend.

REQ 4: SEGURIDAD

  • Implementación: Autenticación segura (JWT/OAuth2), protección contra inyecciones SQL/XSS y registro de auditoría de acciones críticas.
